iter_points(**kwargs)[source]¶

always returns an iterator, this can be overridden in a subclass if it’s more efficient to iterate than a call to gen_points, calling iter_points should make an effort to be the fastest way to iterate through the points, ignorant of caching. In cython, it seems to take twice the time to iterate over a generator, than to use a for loop over indices of an array, but only if the array already exists
